# 桌宠脸部重设计方案

## 目标

在不改动桌宠功能逻辑、窗口行为和状态机的前提下，重做桌宠脸部视觉语言，让当前形象从“符号化 SVG 猫脸”升级为更好看的二次元萌系猫咪角色。

本次目标明确聚焦在以下结果：

- 更像猫咪，而不是几根线拼出的表情
- 更符合 `日系二次元猫咪感`
- 气质偏 `幼萌大眼`
- 整体仍保持清秀、灵气，不做过甜或过俗的风格
- 各状态切换后仍然维持“同一只好看的猫”

## 已确认方向

- 风格方向：`日系二次元猫咪感`
- 脸部改动幅度：`大改`
- 气质方向：`灵气型`
- 眼部倾向：`偏幼萌大眼`
- 推荐方案：`二次元萌系猫脸`

这意味着本次设计不再沿用当前“简笔线条猫脸”的表达思路，而是直接替换为更角色化的猫咪脸部结构。

## 核心问题

当前桌宠“不好看”的根因不是某一个五官参数有偏差，而是整套脸部表达体系存在问题：

- 脸型偏尖，幼态感不够
- 眼睛过于符号化，更像线条表情而不是猫咪角色眼
- 鼻口区过弱，缺少猫咪识别度
- 五官和轮廓组合后整体更像“SVG 图形”，不像可爱的猫咪形象
- 状态切换依赖少量线条变化，容易让脸在某些状态下显得呆、怪或失衡

因此，本次不是“在原脸上继续修修补补”，而是重建一套新的脸部设计基线。

## 设计方案

### 1. 脸型

新脸型采用更圆润的幼态轮廓，但不走过分夸张的圆球脸方向。

设计原则：

- 额头更饱满，提升幼态感
- 面中区域更顺，减少目前偏尖、偏硬的观感
- 下巴更短更收，但保持猫咪角色的清秀感
- 整体接近“圆润的倒水滴脸型”，而不是尖锐三角脸

预期结果：

- 第一眼更可爱
- 更像角色化猫咪
- 不会因为过度圆脸而失去灵气

### 2. 眼睛

眼睛是这次重设计的核心。旧方案中的线条眼将被替换为更像二次元猫咪角色的猫眼结构。

设计原则：

- 明显放大眼睛体量，强化可爱感
- 提升上眼线存在感，让眼神更聚焦
- 眼型保留猫感，不做成人类少女大眼路线
- 保留灵气和陪伴感，不做高冷锐利型
- 通过高光、开合和弧线控制状态变化，而不是只用简化线条替代眼睛

预期结果：

- 眼睛成为整张脸的视觉中心
- 观感更有神、更灵动
- 同时维持清秀感，不会变成低幼表情包风格

### 3. 鼻口区

鼻口区从“弱符号”升级为“明确猫咪识别区”，这是让桌宠更像猫的重要一步。

设计原则：

- 使用更明确的小三角鼻
- 增加短嘴套结构，让鼻子和嘴不再漂浮在脸中央
- 嘴型改为更自然的轻弧猫嘴
- 保持克制，不堆太多拟物细节

预期结果：

- 一眼更像猫咪角色
- 脸部不再空
- 可爱感由结构本身提供，而不是只依赖腮红或表情线

### 4. 耳朵与脸部重心

耳朵保留猫感，但会配合新脸型同步调整，以避免“头脸可爱、耳朵却生硬”的割裂感。

设计原则：

- 耳朵轮廓更干净
- 与头顶弧线衔接更自然
- 通过调整脸部重心，让眼睛位置略上移，提升幼态感和灵气

预期结果：

- 脸部整体更协调
- 不会出现“耳朵是旧版本、脸是新版本”的拼接感

## 表情策略

本次不采用“每个状态画一张不同的脸”，而采用 `统一底脸 + 小幅表情变化` 的策略。

### 基线原则

- 所有状态共用同一套好看的脸型、眼距、鼻口区和头部比例
- 先保证静态脸本身就成立
- 状态变化只发生在细节层，而不是破坏底层结构

### 可变化的内容

- 眼睛开合程度
- 眼型弧度
- 高光与神态细节
- 嘴角轻微变化
- 脸颊和表情氛围的少量调整

### 不做的内容

- 不做夸张扭曲的表情变化
- 不让某些状态的脸退化成简笔表情
- 不让左右眼或嘴型因为状态变化而失衡

### 状态气质基线

- `待机`：灵动、可爱、轻陪伴
- `办公 / 阅读`：更专注，但仍然漂亮
- `生成中 / 思考中`：更认真，不做苦脸
- `摸鱼 / 轻松`：更软一点，但不傻笑

预期结果：

- 所有状态下都还是“同一只好看的猫”
- 不再出现状态一切换就脸崩的问题
- 后续继续迭代表情时也更稳定

## 实现边界

本次属于纯视觉形象升级，只修改桌宠形象绘制层，不改功能逻辑。

### 修改范围

- 桌宠头部轮廓
- 耳朵与脸部布局
- 眼睛、鼻口区、嘴型的基线设计
- 各状态下的表情细节参数
- 必要的局部动画锚点和位置微调

### 不修改范围

- 桌宠开关逻辑
- 拖动逻辑
- 窗口行为
- Rust 侧状态机
- 设置页交互和文案

## 组件落点

结合当前项目结构，本次主要落点在前端桌宠绘制与状态映射层：

- `AvatarCanvas.svelte`
  负责脸部元素的 SVG 呈现与组合，后续会替换旧的五官绘制方式
- `avatarOutline.js`
  负责头部与耳朵轮廓，后续会重画头脸相关路径
- `avatarStateMeta.js`
  负责不同状态下的眼睛、嘴型和神态参数，后续会改成建立在新底脸上的表情变化规则

必要时可以同步微调局部动画锚点，但不会扩散到桌宠窗口逻辑或 Rust 状态判断逻辑。

## 验收标准

- 第一眼明显比当前版本更好看
- 明显像一只二次元猫咪角色，而不是 SVG 拼装表情
- 眼睛成为核心识别点，且具备灵气
- 脸型更圆润，但不显得傻或土
- 鼻口区更像猫，不再空或弱
- 各状态切换后仍然稳定好看
- 不因为表情变化导致脸部失衡、变怪或变呆

## 测试与验证

本次虽然是视觉优化，但仍需要最基本的结构验证，避免改脸时引入回归。

验证重点：

- 现有桌宠轮廓测试需要同步更新，确保新路径仍满足当前布局边界
- 状态元数据测试需要覆盖新的默认脸部参数和状态差异
- 手动检查待机、办公、阅读、生成中等关键状态，确认不会出现脸部错位或表情崩坏

建议验证方式：

- `node --test src/lib/components/Avatar/avatarOutline.test.js`
- `node --test src/lib/components/Avatar/avatarStateMeta.test.js`
- 必要时补充 `avatarAppearance` 相关验证，确保视觉重构未误伤现有模式映射

## 风险与控制

### 风险 1：改得更像角色后，失去原桌宠识别度

控制方式：

- 保留当前猫咪桌宠的基础身份，不改物种、不改整体角色设定
- 仅重做脸部视觉语言，不把它变成完全不同的角色

### 风险 2：大眼化后变得低幼或廉价

控制方式：

- 用“更精致的眼型”而不是“无限放大眼睛”来实现可爱
- 保持清秀轮廓和克制嘴型，不走甜腻路线

### 风险 3：状态表情一多，脸部再次崩坏

控制方式：

- 采用统一底脸策略
- 把状态变化限制在细节层，不重画整张脸

## 结论

本次桌宠优化不应再停留在“调一调眼睛曲线”这种局部修补上，而应该建立一套新的猫咪脸部设计基线。

最终目标不是“让现在这张脸没那么丑”，而是让用户第一眼就觉得：

“这是一个好看的、可爱的、灵气的二次元猫咪桌宠。”
